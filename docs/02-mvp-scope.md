# MVP機能一覧 — 飲食店マーケティングOS

本ドキュメントは、開発フェーズごとの機能スコープを定義する。Phase 1（MVP）で価値検証に必要な最小構成を提供し、Phase 2・3 で段階的に拡張する。

---

## 1. フェーズ方針
- **Phase 1（MVP）**: コンサルティングの中核フロー（クライアント → 店舗 → ヒアリング → 診断 → 課題 → KPI → 施策 → タスク → 月次レポート）を一気通貫で提供する。データ層はモックデータ層で動作させ、UI/UXとフローを確立する。
- **Phase 2**: 分析系（市場・商圏・競合・需給）、戦略設計、SNSコンテンツ、ミーティング、ナレッジ、ファイル管理を追加する。
- **Phase 3**: AI支援、外部連携、クライアント専用画面、レポート自動送信、通知など高度化・自動化を行う。

---

## 2. Phase 1（MVP）機能

| 機能 | 概要 |
| --- | --- |
| 認証 | Supabase Auth によるログイン／ログアウト、セッション管理、ロール判定。 |
| ダッシュボード | 横断サマリー。注意店舗、期限超過タスク、KPI進捗、担当クライアント一覧。 |
| クライアント管理 | クライアントの登録・一覧・詳細・編集・論理削除。契約情報の基本管理。 |
| 店舗管理 | 店舗の登録・一覧・詳細・編集。クライアントへの紐づけ。 |
| ヒアリング | 初回ヒアリングシートの作成・保存・閲覧。 |
| 店舗診断 | 5つの価値カテゴリに基づく診断項目スコアリングとレーダーチャート表示。 |
| 課題管理 | 課題の抽出・登録・優先度管理。診断・戦略との紐づけ。 |
| KPI管理 | KPI定義と月次実績入力、目標対比の可視化。 |
| 施策管理 | 施策の登録・進捗管理。KPI・課題との紐づけ。 |
| タスク管理 | タスクの作成・担当割当・期限・ステータス管理。 |
| 月次レポート | レポートの作成・編集・承認・PDF出力（手動下書き）。 |
| 基本的な権限管理 | admin / marketer のロールに基づく画面・操作制御。ユーザー一覧。 |

## 3. Phase 2 機能

| 機能 | 概要 |
| --- | --- |
| 市場・商圏分析 | 市場規模・トレンド、商圏人口・動線などの記録と可視化。 |
| 競合分析 | 競合店舗リスト、強み弱み、ポジショニング比較。 |
| 需要・供給分析 | 需要（ニーズ・来店動機）と供給（キャパ・提供体制）の分析。 |
| 戦略設計 | 課題を踏まえた戦略の立案・登録、課題・KPI・施策との紐づけ。 |
| SNSコンテンツ管理 | SNS/MEO/LINE等の投稿企画・下書き・投稿予定・投稿済み管理。 |
| ミーティング管理 | 打合せ記録、議事録、アクションアイテム管理。 |
| ナレッジ管理 | 成功／失敗事例のナレッジ登録・タグ検索・再利用。 |
| ファイル管理 | 資料・画像・PDFのアップロードと各エンティティへの紐づけ。 |

## 4. Phase 3 機能

| 機能 | 概要 |
| --- | --- |
| AI機能 | 診断コメント・課題抽出・戦略案・月次レポート下書きのAI生成（人が承認）。 |
| 各種外部連携 | Google Business Profile(MEO)、SNS各種API、広告媒体、予約システム等との連携。 |
| クライアント専用画面 | クライアント（client ロール）向けの閲覧専用ポータル。 |
| レポート自動送信 | 月次レポートの承認後、クライアントへの自動送付。 |
| 通知 | KPI未達・期限超過・承認依頼などのアプリ内／メール通知。 |

## 6. MVPのゴール（完了の定義）
- 1人の担当者が、1クライアント・1店舗について、登録 → ヒアリング → 診断 → 課題 → KPI → 施策 → タスク → 月次レポートまでをシステム上で完結できる。
- ダッシュボードで担当店舗の状態（注意店舗・期限超過タスク）が一覧できる。
- admin / marketer のロールで画面・操作が適切に制御される。
- モックデータ層で全画面が動作し、後から Supabase/Prisma に差し替え可能な構造になっている。
