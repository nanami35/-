# 飲食店マーケティングOS

飲食店の集客・売上向上を支援するマーケティング会社向けの、**社内業務管理システム**です。
見込み顧客の管理から、店舗分析・戦略立案・施策実行・効果検証・月次レポートまでを一元管理し、
業務の属人化を防ぎ、誰が担当しても一定品質で支援を進められる状態を目指します。

> 飲食店の現状分析 → 戦略設計 → 施策実行 → 効果検証 → 改善提案 を一貫管理する「飲食店マーケティングOS」

## 技術構成

- **フレームワーク**: Next.js 15（App Router）+ React 19
- **言語**: TypeScript（strict / `noUncheckedIndexedAccess`）
- **スタイル**: Tailwind CSS（shadcn/ui 準拠の自作 UI コンポーネント）
- **グラフ**: Recharts（レーダー / 折れ線 / 棒）
- **フォーム**: React Hook Form + Zod（バリデーション）
- **DB**: PostgreSQL（Supabase）
- **ORM**: Prisma
- **認証**: Supabase Auth（デモは Cookie ベースの簡易認証）
- **配色**: 濃紺 `#1A2B4A` / ゴールド `#C9A227`

## Supabase / データベース接続手順

1. [Supabase](https://supabase.com/) でプロジェクトを作成。
2. `.env.example` を `.env` にコピーし、`DATABASE_URL` / `DIRECT_URL` / `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` を設定。
3. スキーマを反映:
   ```bash
   npm run db:generate   # Prisma Client 生成
   npm run db:push       # スキーマを DB に反映（開発用）
   ```
4. サンプルデータを投入:
   ```bash
   npm run db:seed
   ```
5. **Row Level Security (RLS)** を有効化し、`organization_id` によるテナント分離ポリシーを設定
   （詳細は `docs/08-implementation-notes.md` / `docs/05-er-diagram.md` を参照）。

> データアクセスは `src/lib/data.ts` に集約されています。
> 本番移行時はこの層の内部を Prisma/Supabase クエリへ差し替えることで、UI を変更せず移行できます。

## 開発ルール

- TypeScript の型を厳密に使用（`any` を避ける）
- コンポーネントは適切に分割し、UI は `components/ui` に再利用可能な形で集約
- 定数は `lib/constants.ts` に集約（ハードコードを避ける）
- **ローディング・空状態・エラー状態**を実装（`loading.tsx` / `EmptyState` / `error.tsx`）
- フォームは Zod でバリデーション
- モバイル表示でも崩れないレスポンシブ設計（業務利用は PC 優先）
- 不要な依存を増やさない

## セキュリティ方針

- 認証必須 / ロールベース権限管理（RBAC）
- `organization_id` によるテナント分離（マルチテナント）
- Supabase Row Level Security を前提
- 入力値バリデーション / 秘密情報はフロントに露出しない
- 操作ログ（`activity_logs`）/ 重要データは論理削除（`deleted_at`）/ 削除時は確認モーダル
